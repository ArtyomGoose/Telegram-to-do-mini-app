# Проект: Telegram Mini App с контролем доступа

## Описание

Telegram Mini App (TWA) на React + Vite для управления ежедневными задачами с синхронизацией в реальном времени через Firebase. Возможности:
- Управление задачами с отслеживанием выполнения
- Прогресс-бары за день и за год (русская локализация)
- Контроль доступа: аутентификация по белому списку Telegram ID
- Синхронизация в реальном времени между устройствами
- Тёмная тема Telegram

## Технологический стек

- **Фронтенд:** React 18, Vite 7
- **Бэкенд:** Firebase Realtime Database
- **Стили:** CSS с переменными темы Telegram
- **Локализация:** Русский язык (даты, текст UI)

## Основные функции

### 1. Контроль доступа (белый список)

**Файл:** `twa/src/auth.js`

Три сценария аутентификации:

1. **Telegram Mini App:** ID пользователя извлекается из `window.Telegram.WebApp.initDataUnsafe.user.id` и проверяется против белого списка мгновенно.
2. **Браузер + сохранённый ID:** Если пользователь посещал приложение раньше, его ID извлекается из `localStorage.getItem('twa_browser_id')` и проверяется.
3. **Браузер + первое посещение:** Показывается форма входа, пользователь вручную вводит свой Telegram ID, затем проверяется против белого списка.

**Белый список:** В настоящее время захардкожен в `auth.js` как `ALLOWED_IDS = ['123456789', '987654321']`. Замените на реальные ID.

**Статусы аутентификации:**
- `null` → Проверяется статус
- `'allowed'` → Пользователь может использовать приложение
- `'denied'` → Показывается страница AccessDenied
- `'browser_login'` → Показывается форма входа

### 2. Управление задачами

**Файл:** `twa/src/App.jsx`

- Добавление задач: `addTask(text)` → сохраняет в Firebase под путём `users/shared_user/tasks/{id}`
- Выполнение задач: `completeTask(id)` → удаляет из списка, увеличивает счётчик `completedToday`
- Все авторизованные пользователи видят один общий список задач (захардкожен `userId = 'shared_user'`)

### 3. Обнаружение смены дня

Запускается при монтировании приложения и каждые 60 секунд:
- Сравнивает сохранённую дату (`localStorage.twa_last_date`) с текущей датой
- Если даты отличаются: помечает все невыполненные задачи как `carriedOver: true` (отображаются красным)
- Сбрасывает счётчик `completedToday` на 0

### 4. Прогресс-бары

**Файл:** `twa/src/components/DateBlock.jsx`

- **Прогресс года:** Дни, прошедшие с начала года / дни в году (учитывает високосные годы)
- **Прогресс дня:** Выполненные задачи сегодня / всего задач начато сегодня
- Форматируется как "Пятница, 27 февраля" (русская локализация)

### 5. Синхронизация через Firebase

**Файл:** `twa/src/firebase.js`

- Слушатели в реальном времени для задач и счётчика выполнения
- Структура данных:
  ```
  users/
    shared_user/
      tasks/
        {id}: { id, text, createdAt, carriedOver }
      completedToday: {number}
  ```
- Слушатели запускаются только после подтверждения аутентификации

## Разработка

### Требования
- Node.js 16+
- Firebase проект с Realtime Database
- Telegram Bot (для интеграции Mini App)

### Установка

```bash
cd twa
npm install
npm run dev      # Запустить dev сервер на http://localhost:5173
```

### Сборка

```bash
npm run build    # Создаёт папку dist/
npm run preview  # Просмотр production сборки локально
```

## Развёртывание

1. Запустите `npm run build` в директории `/twa`
2. Разместите содержимое папки `dist/` на статическом хостинге (например, Vercel)
3. Зарегистрируйте URL в BotFather для вашего Telegram бота

## Конфигурация

### Учетные данные Firebase

Отредактируйте `twa/src/firebase.js` для обновления конфига Firebase при необходимости.

### ID в белом списке

Отредактируйте `twa/src/auth.js`:
```js
export const ALLOWED_IDS = ['YOUR_ID_1', 'YOUR_ID_2']
```

## Важные заметки

- **Общий список задач:** Все авторизованные пользователи видят и редактируют один и тот же список задач. Чтобы переключиться на данные отдельного пользователя, измените `userId` с `'shared_user'` на ID аутентифицированного пользователя.
- **Ключи localStorage:**
  - `twa_last_date` — Дата последнего открытия приложения (YYYY-MM-DD)
  - `twa_browser_id` — Введённый пользователем Telegram ID (только браузер)
- **Telegram SDK:** Загружается синхронно в `index.html` перед другими скриптами. Опциональный оператор (`?.`) защищает от отсутствия SDK в обычном браузере.

## Принадлежность файлов

- **Файлы Python бота** (`bot.py`, `test_api.py`) — Не трогаются TWA. Управляются отдельно.
- **Файлы TWA** (директория `twa/`) — Основной фокус. Вся логика аутентификации/задач/синхронизации находится здесь.

## Поведение UI по платформам

### Telegram Mini App
- Автоматическая проверка ID пользователя из SDK
- Страница AccessDenied показывается без кнопки повтора (состояние только для чтения)
- Форма входа не требуется

### Режим браузера
- Форма для ручного ввода ID при первом посещении
- ID сохраняется в localStorage для последующих посещений
- Кнопка "Попробовать снова" на странице AccessDenied
- Возможность попробовать другие ID
